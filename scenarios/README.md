# scenarios/

Each subdirectory is one province-focused scenario. Layout convention:

```
scenarios/<province>/
├── scenario.yaml            # single config; province-agnostic schema
├── inputs/                  # generated by tools/build_scenario.py — DO NOT hand-edit
│   ├── modules.txt          # generated, per-scenario module list
│   └── *.csv                # sliced subset of base inputs/
├── outputs/                 # generated by switch solve — .gitignored (large)
├── logs/                    # generated by switch solve --log-run — .gitignored
└── results_archive/         # selectively committed:
                             #   - summary CSVs (capacities, dispatch, costs)
                             #   - calibration reports
                             #   - charts (PNG/PDF)
```

## `scenario.yaml` schema

```yaml
name: shanghai                 # arbitrary identifier
focal_zone: Shanghai           # primary zone of analysis
adjacent_zones: [Jiangsu, Zhejiang, Anhui, Fujian]   # neighbor zones modeled in full
uhv_partner_zones: [Sichuan]   # long-distance UHV partners modeled in full
external_power_mode: endogenous   # endogenous | exogenous | hybrid
national_caps:
  total_capacity_limits: prorate_by_demand   # drop | keep | prorate_by_demand
  carbon_policies: prorate_by_demand
period_start: 2020
period_end: 2060
modules_extra: [cn_modules.tech_plans]
modules_disable: []
```

## Running

```bash
# build scenario inputs (slices base inputs/ into scenarios/<name>/inputs/)
python tools/build_scenario.py --scenario scenarios/<name>/scenario.yaml

# verify it constructs the model
python tools/validate_scenario.py --inputs-dir scenarios/<name>/inputs

# solve
switch solve \
  --inputs-dir scenarios/<name>/inputs \
  --outputs-dir scenarios/<name>/outputs \
  --solver appsi_highs \
  --solver-options-string "solver=ipm run_crossover=off" \
  --log-run --logs-dir scenarios/<name>/logs
```

## Province-agnostic guarantee

Migrating to a new province requires **only** creating a new
`scenarios/<new-province>/scenario.yaml`. Zero Python code edits. This is
enforced by Phase F's acceptance test.
